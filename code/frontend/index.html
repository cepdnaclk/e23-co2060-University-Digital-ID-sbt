System Features

A realistic MVP flow for student identity management

This prototype demonstrates the pages and workflows required for student registration, wallet login, admin actions, token requests, QR verification, and complaint handling.

🎓

Student Registration

Students submit basic details and connect a wallet to their profile.
🔐

Wallet Login

Authentication is designed around wallet verification instead of password storage.

🪪

Soulbound Token

Admin issues a non-transferable ERC721 identity token to the student wallet.
📱

QR Verification

Students can generate a QR preview for identity verification workflows.
